Eric's Timestamp microservice

Required user stories:
    1) I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016)
    2) If it does, it returns both the Unix timestamp and the natural language form of that date.
    3) If it does not contain a date or Unix timestamp, it returns null for those properties.

Example usage:

After this web address, type a Unix timestamp or a natural language date. For example, try these links:

  • /December 15, 2015 (valid date)
  • /1480807900 (valid Unix timestamp)
  • /December 32, 2012 (invalid date)
  • /hamburger (invalid date)

Example output:

Your browser should display the resulting JSON, so if you typed in /1450155600 you would see something like this:


{"unix":"1450155600","natural":"Tuesday, December 15th 2015"}
